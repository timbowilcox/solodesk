## Skill authoring rules (from CLAUDE.md, repeated for emphasis)

1. Single responsibility. One job per skill.
2. Inputs and outputs explicit.
3. Adversarial counterpart authored at the same time, or workflow rationale documented.
4. Anti-patterns listed.
5. Budget set (tokens + cents).
6. Think-like-the-agent exercise completed before merge.

## How a skill gets added

When a sprint ships a new skill:
1. Create `.claude/skills/<n>/SKILL.md` with full frontmatter.
2. Add row to this index.
3. Reference rubric in `.claude/rubrics/<skill-name>.md`.
4. Add invocation handler in `/lib/skills/<n>.ts`.
5. Wire to whichever route handler or cron triggers it.
6. Log invocation to `loop_runs` table — non-negotiable.
